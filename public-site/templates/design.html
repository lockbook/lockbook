Design

Design Type

Light

surfaces lift · light appears

  • Interactive surfaces lift on hover and a light effect appears alongside. The rise and the light read as one gesture; press snaps the surface flat instantly. Animation: 0.3s cubic-bezier(0.22, 1, 0.36, 1) springy ease-out.
  • Shadow — drop below the surface, for elevating cards, panels, and other generic surfaces. Tokens --shadow-y, --shadow-blur, --shadow-alpha (alpha jumps 0.2 → 0.8 in dark mode).
  • Glow — colored halo + white-hot core under the surface, the marquee button effect. Productized as the .btn-glow wrapper recipe (shadow-gradient pseudo + plus-lighter core pseudo).
  • Source of truth for glow: --z (button height above page, px). State machine via --btn-z-rest / --btn-z-hover — defaults 0 / 3 (secondary); primary CTAs opt in to 3 / 5 (glow at rest).

Shadow · surface elevation

Flat at rest. Hover lifts by --shadow-y while the shadow drops by the same amount. For generic surfaces; for buttons, use the glow recipe below.

box-shadow: 0 var(--shadow-y) var(--shadow-blur) rgba(0, 0, 0, var(--shadow-alpha));

Color

from the Mnemonic theme

  • Neutrals + one accent + paired chroma palette.
  • Foreground is never colored — except in monospace.
  • Proportional text stays ink; font-mono may take a chroma color where the type reads as data, label, or code.

Three layers

  • Every color sits at one of three layers — primitive, family, role.
  • Verbal shorthand mirrors the code: "dark blue" = Layer 1, "foreground blue" = Layer 2, "accent text" = Layer 3.
Layer What it is Naming Use when
1 · Primitive Literal hex, immutable. Named after the shade. chroma-blue · chroma-sky · 10 more You need a specific shade regardless of mode (a 3rd-party brand color match; a code-block syntax token).
2 · Family Semantic, mode-aware. A hue with a foreground variant (readable on the page) and a background variant (brand-stable). fg-blue (flips) · bg-blue (constant) You want "the foreground blue" without thinking about mode. Today only blue has a family; other chromas grow one when they earn it.
3 · Role Bound to a UI pattern. Thin alias over Layer 2. accent = bg-blue · accent-fg = fg-blue Default — the role name says what for, not which color. Most callsites should reach for this layer.

Spine

  • Six mode-flipping CSS vars driving every neutral surface + text.
  • Foreground family: bg-fg / bg-fg-muted / bg-fg-faint (plus text-* and outline-* counterparts).
  • Background family: bg-line / bg-raised / bg-base (plus text-* and outline-*).

Buttons

flat · wireframe · glass

  • Three intent classes, one shared hover.
  • Flat (.btn-flat) — identity in the fill.
  • Wireframe (.btn-primary + .btn-segmented) — identity in the outline.
  • Glass (.btn-glass) — identity through translucency; for buttons over imagery.
  • All three pair with a .btn-glow wrapper for the lift-and-underglow hover.
  • Modifiers: .is-accent swaps the identity to Steel Blue; .is-selected holds the hover state without the pointer.

Primary CTA · glow-as-affordance

  • Surface is bg-base (matches page background); no fill, no outline.
  • The glow at z=3 rest IS the affordance — the button reads as a button because it's hovering above the page with a colored halo underneath.
  • Hovers to z=5 (brighter glow), presses to z=0 (snaps flat — the glow vanishes momentarily).
  • For the page's primary action: hero CTA, download buttons, anything that's "THE" thing to do on that view.

Wave field

tiles · waves · spring

  • Ambient brand effect for hero / landing surfaces — motion in place of imagery.
  • Same 2:1 horizontal rhombus shape as the book logo, tessellated edge-to-edge across a fixed-aspect container.

Mechanics

  • Each tile carries a signed displacement u (px). Per-frame leapfrog integrates the damped 2D wave equation: u_new = (2−γ)u − (1−γ)u_prev + c²·Σ(u_nb − u), summed over the 4 edge-sharing diagonal neighbors. The iso-rhombus grid is topologically a square grid at 45° + 2:1 squish — same-row tiles only touch at corners and aren't true neighbors.
  • Cursor adds a damped-spring force k·(target − u) − b·(u − u_prev) to tiles in radius, NOT a hard clamp — so velocity history stays intact across cursor sweeps and the wave eq sees consistent state on entry and exit.
  • Displayed --lift is the 2-tap moving average (u + u_prev) / 2 — kills the Nyquist checkerboard mode that leapfrog has zero dissipation for. u_new is hard-clipped to ±--lift-max so cursor resonance can't pump tiles off-screen.
  • Initial impulse (page load or button) seeds u = max·fall², u_prev = 3u — peak displacement plus impact velocity (kinetic + potential), like a stone still moving when it hits water.
  • Stability requires c² < ~1/4 on the 4-neighbor laplacian; settings sit well under for slow, smooth propagation. Idle field early-returns from the rAF loop; offscreen fields pause via IntersectionObserver; prefers-reduced-motion disables the sim entirely.

A centered drop fires 600 ms after layout. Move the cursor over the field for live interaction. Sliders below tune the running sim; the re-drop button replays the impulse.
